# Nunki Preliminary Requirements (WIP)

## Application Types
* Capabilities to be invoked as web application or CLI application

## Backend Capabilities
* Support for zmq, opcua client/server, REST client, kafka
* Support for mongodb data backends
* Support for scalability, and dockerization
* High level of code quality, requirement traceability and tests coverages.
* Uber jar creation, source jar creation
* Support for Graalvm JDK and native build.
* Authentication support with configurable backends among: hardwired identifiers, Kerberos and LDAP
* Support for websocket/STOMP async exchanges

## Scripts & Helpers
* Bash based helper for tests, offline build, integration tests with quasar project (to be defined)

## Frontend Capabilities (Svelte)
* Svelte frontend
* ability to display dynamic SVG diagrams and synoptics,
* ability to display graph of time series
* ability to display plantuml diagrams
* ability to use tree views and dynamic tables
* Support for authentication
* Offline build support if possible
