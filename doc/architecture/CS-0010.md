# Constraint 0010

## Title

Project's quality standards

## Description

This set of constraints defines the general quality standards for the project. 


- [CS-0010.1] All code shall be associated to a requirement, a feature request, a task or a bug fix.
  - [CS-0010.1.1] Each class, methods, field, control structure, consistent block of statements shall be associated to a requirement, a feature request, a task or a bug fix.
  - [CS-0010.1.2] Association of code to a requirement, a feature request, a task or a bug fix shall be done using a comment block with the reference of associated requirement, feature request, task or bug fix.
- [CS-0010.2] Every requirement, feature request, task or bug shall have a unique structured reference usable in any code or documentation file.
  - [CS-0010.2.1] Requirements reference format shall be structured as follow (regex): REQ-[0-9\.]{5}
  - [CS-0010.2.2] Feature request reference format shall be structured as follow (regex): FR-[0-9\.]{5}
  - [CS-0010.2.3] Task reference format shall be structured as follow (regex): TSK-[0-9\.]{5}
  - [CS-0010.2.4] Bug reference format shall be structured as follow (regex): BUG-[0-9\.]{5}
  - [CS-0010.2.5] Bugfix reference format shall be structured as follow (regex): FIX-[0-9\.]{5}
